# Security Policy

## Reporting a Vulnerability
Please report security issues privately to the repository owner before public disclosure.

Include:
- Affected file/module
- Reproduction steps
- Impact
- Suggested fix (if available)

## Disclosure Guidelines
- Do not publish exploit details before a fix is available.
- Allow a reasonable remediation window before disclosure.

## Scope
This project interacts with privileged disk/security tooling. Any issue affecting command execution, privilege handling, or password processing should be treated as high priority.
